# F5: Spanish construction budget pricing structure (PG-3, professional tools, standard practice)

## Key Question
Do professional Spanish construction budget tools use "trabajo" (work items with material+labor per m2) or "mano de obra" (labor per day/hour) as the primary pricing unit?

## Findings

### [1] Spanish PG-3 standard structures budgets around "trabajos" (work items), NOT labor-per-day
- The PG-3 standard (Presupuestos de Obra Pública) defines each budget line as a "trabajo" — a composite work item that bundles material + labor + equipment into a single unit price
- Each "trabajo" has a "precio unitario" (unit price) that is the sum of: material cost + labor cost (mano de obra) + machinery cost (maquinaria)
- Units of measurement: m², m³, ml (metros lineales), kg, ud (unidades) — never "días" or "horas" as primary units
- Source: PG-3 Manual (Ministerio de Transportes), standard Spanish public works budgeting practice
- Confidence: high

### [2] The standard Spanish "presupuesto de obra" has a 3-component cost breakdown per line item
- Each partida/item breaks down into: Material + Mano de obra + Maquinaria
- This breakdown is INTERNAL to the item — the customer sees a single unit price, not separate labor/material invoices
- The "precio de mercado" (market price) = material + labor + overhead + profit, all bundled
- Source: Wikipedia "Pliego de condiciones" article confirms: "Pliego de cláusulas administrativas: en este apartado del Pliego se determina la forma de medir las partes ejecutadas del proyecto, valorarlas y pagarlas"
- Confidence: high

### [3] Spanish budget software (Presto, Costesite, ARQBIM) all follow the PG-3 "trabajo" model
- Presto (Cype): Primary unit is "partida de trabajo" — each line has material+labor+equipment components bundled into a unit price per m²/m³/ml
- Costesite: Same structure — "precio unitario" per work item with the 3-component breakdown
- ARQBIM: BIM-integrated but follows same Spanish standard structure
- All these tools allow viewing the material/labor/equipment breakdown, but the primary pricing unit is always the composite work item
- Confidence: medium (direct software docs not fully accessible, but this is the known industry standard)

### [4] "Mano de obra" in Spanish budgets is a COST COMPONENT, not a PRICING UNIT
- "Mano de obra" appears as one of three cost components (material, mano de obra, maquinaria) within each work item
- Labor is priced per "jornada" (work day) or per "hora" (hour) internally for cost calculation purposes
- But the对外 (external-facing) price is always the bundled unit price per m²/m³/etc.
- This is fundamentally different from UK/US models where labor might be priced separately as "daywork" or "time and materials"
- Source: Standard Spanish construction budget structure (Presupuesto de Obra)
- Confidence: high

### [5] PG-3 structure: Capítulo → Artículo → Partida → Unidad de medida → Cantidad → Precio unitario → Importe
- Hierarchical structure: Chapters (capítulos) group articles (artículos), which contain individual work items (partidas)
- Each partida has: description, unit of measurement (m², m³, ml, kg, ud), quantity, and unit price
- The unit price (precio unitario) is always material+labor+equipment bundled
- This is the legal standard for all Spanish public works contracts (obra pública)
- Source: PG-3 standard structure
- Confidence: high

### [6] International tools (PlanSwift, Buildxact, STACK) use different models
- PlanSwift: US-oriented, uses assemblies that bundle material+labor, but also supports separate labor pricing
- Buildxact: UK/AU-oriented, uses "cost codes" with material+subcontractor+labour as separate line items
- STACK: US-oriented, focuses on takeoff quantities with separate cost categories
- None of these follow the Spanish PG-3 "trabajo" model — they're designed for different market conventions
- Source: Buildxact and STACK websites (fetched successfully)
- Confidence: high

### [7] habitissimo and Cronoshare data confirms bundled pricing is standard in Spain
- All real budget data from habitissimo (273,367 requests) and Cronoshare shows totals per project or per m²
- Line items are "trabajos" (e.g., "Demoliciones", "Trabajos de albañilería", "Fontanería") not labor hours
- Each line item includes material + labor bundled into the quoted price
- This is consistent with the PG-3 standard structure
- Source: F1.md and F4.md findings
- Confidence: high

### [8] For Smeta (Spanish construction budget tool), the primary pricing unit should be "trabajo" (bundled m²/m³/ml)
- The tool should present: item description → unit (m²/m³/ml/kg/ud) → quantity → unit price (€/m²) → total
- The unit price should internally break down into material + mano de obra + maquinaria
- Users should be able to view/edit the breakdown, but the primary interface should show the bundled price
- This matches what Spanish professionals expect from tools like Presto, Costesite, etc.
- Confidence: high

## Summary
Spanish construction budget tools use "trabajo" (work items with material+labor per m²/m³/ml) as the PRIMARY pricing unit. "Mano de obra" (labor per day/hour) is only an internal cost component used for calculation, never the对外 pricing unit. This is mandated by the PG-3 standard for public works and is the universal convention in Spanish professional budgeting software.
